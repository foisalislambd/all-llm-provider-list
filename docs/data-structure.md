# Data Structure

All machine-readable data lives in the `data/` folder.

## File overview

```
data/
├── providers.json       # 97 providers — metadata & endpoints
├── models.json          # Model catalogs (generated by scripts/sync_models.py)
└── static_models.json   # Manual fallback model lists
```

## providers.json

Array of provider objects:

| Field | Type | Description |
|-------|------|-------------|
| `id` | number | Unique ID (1–97) |
| `name` | string | Display name |
| `slug` | string | URL-safe key for lookup |
| `aliases` | string[] | Alternative search names |
| `category` | string | Frontier, IaaS, Gateway, Local, etc. |
| `website` | string | Official URL |
| `api_base_url` | string | Standard API endpoint |
| `popular_models` | string[] | Highlighted models |
| `env_variable` | string \| null | `.env` key name |
| `openai_compatible` | boolean | Works with OpenAI SDK? |
| `notes` | string | Short integration note |

Example:

```json
{
  "id": 20,
  "name": "Groq",
  "slug": "groq",
  "aliases": [],
  "category": "IaaS",
  "website": "https://console.groq.com",
  "api_base_url": "https://api.groq.com/openai/v1",
  "popular_models": ["llama-3.3-70b-versatile"],
  "env_variable": "GROQ_API_KEY",
  "openai_compatible": true,
  "notes": "LPU hardware; extremely fast TTFT"
}
```

## models.json

Generated by `scripts/sync_models.py`. Structure:

```json
{
  "updated_at": "2026-06-16T...",
  "total_providers": 97,
  "total_models": 945,
  "providers": {
    "groq": {
      "provider_id": 20,
      "provider_name": "Groq",
      "category": "IaaS",
      "api_base_url": "https://api.groq.com/openai/v1",
      "models_endpoint": "https://api.groq.com/openai/v1/models",
      "count": 10,
      "source": "static",
      "updated_at": "...",
      "models": ["llama-3.3-70b-versatile", "..."]
    }
  }
}
```

### `source` values

| Value | Meaning |
|-------|---------|
| `live-public` | Fetched from public API (no key) |
| `live-auth` | Fetched with API key from env |
| `static` | From `static_models.json` |
| `unavailable` | No models found |

## static_models.json

Manual catalog used when live fetch fails:

```json
{
  "providers": {
    "openai": ["gpt-5.5", "gpt-4o", "o3-mini"],
    "groq": ["llama-3.3-70b-versatile"]
  }
}
```

## Scripts that use this data

| Script | Reads | Writes |
|--------|-------|--------|
| `llm_lookup.py` | providers.json, models.json | — |
| `scripts/sync_models.py` | providers.json, static_models.json | models.json |

## README vs JSON

- **README.md** — human-readable tables for browsing on GitHub
- **data/*.json** — source of truth for scripts and automation

When updating data, change JSON first, then update README tables to match.
